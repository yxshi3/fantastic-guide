minimalist theme (HTML & css & js)

  • THE CHALLENGE

I have received multiple compliments and requests to publish the current design of my website as a theme. I recognized that it is often hard when you are trying to code your first personal site because you are overwhelmed by the number of things that you need to consider and don't know where to start. So I hope by releasing this theme on GitHub with inline code comments, more people could find coding their first personal site or portfolio easier.

After taking the web.lab in winter 2021, I decided to apply HTML, CSS, JavaScript, and UI/UX knowledge that I learned by redesigning my old personal website. In comparison to version 1.0, I wrote all of my code instead of using a bootstrap framework. Coding my website from scratch has helped me to have a better understanding of web development and to improve my programming skills in terms of readability. If you found a bug or have a feature request, feel free to submit your issues.

I'm working hard to make this theme available as soon as possible. Check back soon or view my current progress!



daq web app (Python & REACT & Typescript & Google Firebase)

  • THE CHALLENGE
  • OUR SOLUTION
  • View On GitHub

Our goal of creating a new DAQ app interface is to build an easily accessible application to store and parse logs for mechanical, simulation, and controls engineers at MIT Motorsports. In comparison to how we have previously handled the log files, the new DAQ app also hopes to improve the general performance of the DAQ app. Besides, it has additional helpful features that could better support our engineers' work.

Built with my teammates on the DAQ team, our DAQ web app is a functional full-stack pipeline that acts as our team's internal "Dropbox" while parses uploaded logs. My responsibility includes implementing log splicing/merging and automatic unit scaling using Python, building portion of the backend storage using Google Firebase, and part of the frontend user interface using React and TypeScript. Through this project, I got introduced to technologies including React, TypeScript, Google Firebase, and Pandas Library. It was fun to learn these new skills by doing a hands-on project. So far, we have completed the MVP. Our next step is to continue adding more features to our web interface and continue maintaining the website.



WRIST DISTRACTOR

  • THE CHALLENGE
  • OUR SOLUTION
  • view the prototype

We want to improve the current methods using Kirschner wires (K-wires) for stabilizing the distal fragments during open-reduction and internal fixation (ORIF) because surgeons sometimes find that the use of K-wires is not worth the potential complication and interference in the workflow. Thus, to aid surgeons in completing ORIF, there is a need for a device capable of concurrently applying traction, reducing, and temporarily securing the fragments of a distal radius fracture into position. This wrist distractor device hopes to help surgeons screw the hardware into place with little effort and greater accuracy.

I worked with Prof. Slocum, Dr. Connor, Dr. Rojas, and Dr. Slocum Jr., to find ways to simplify and improve the previous iteration of our wrist distraction device. I also contributed to designing our current iteration in Fusion 360 and SolidWorks, and then prototyped the device in Prof. Slocum's basement. Working with Prof. Slocum, I learned the fundamentals of design principles and how to use computer-aided design and prototyping tools such as lathe and mill. Our work currently undergoes technology licensing review, and soon will be patent.
